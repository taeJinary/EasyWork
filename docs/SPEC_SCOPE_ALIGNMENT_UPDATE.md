# TaskFlow V2 확장 범위 정합성 업데이트

작성일: 2026-03-02  
적용 브랜치 기준: `develop`

## 1. 목적
- 기존 V2 문서에서 "범위 제외"로 정의된 항목을 구현 대상에 포함하기로 한 결정 사항을 고정한다.
- 구현 우선순위를 재정렬해 이후 PR의 기준 문서로 사용한다.

## 2. 범위 변경
아래 항목을 V2 확장 구현 범위로 포함한다.

1. `OAuth` 로그인
2. 이메일 초대 발송
3. 첨부파일 업로드
4. 멘션 알림
5. 푸시 알림

## 3. 우선순위
아래 순서를 현재 기준 실행 우선순위로 확정한다.

1. 문서 정합성 업데이트
2. 품질 안정화 (`통합 테스트 정리`, `WebSocket 복원력 개선`, 인코딩 정리)
3. `OAuth` 로그인
4. 이메일 초대 발송
5. 첨부파일 업로드
6. 멘션 알림
7. 푸시 알림

## 4. 작업 단위
각 우선순위는 독립 PR 단위로 진행한다.

- `P1`: 문서 정합성 업데이트
- `P2`: 품질 안정화
- `P3`: OAuth 로그인
- `P4`: 이메일 초대 발송
- `P5`: 첨부파일 업로드
- `P6`: 멘션 알림
- `P7`: 푸시 알림

## 5. 운영 규칙
- 프론트엔드 구현 시작 전 사용자에게 사전 보고한다.
- `Task.md`에는 PR 링크/브랜치 정보 없이 구현 내용만 기록한다.
- `Task.md` 항목은 `작성시각`을 포함한다.

## 6. 참고
- 본 문서는 기존 V2 원문을 대체하지 않고, 현재 프로젝트 실행 기준을 보강하는 오버레이 문서다.
